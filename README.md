# TinyUrl — the B2B project for making short links.

### Driver’s Payment problems

1. People are not gonna pay just only for “people want shorted URLs”.
2. The URL is just the transport layer. The business is not about creating alternatives solutions with more beautiful UI design or more smooth animations. That’s about the payment’s driver — we must understand that URL is ecosystem and is only thing that we need to do solutions over ecosystem. 

### Then who is gonna pay for it?

Nobody :) 

For a **System Design** project, a URL shortener is popular because it naturally introduces interesting engineering challenges:

- generating unique short IDs,
- handling billions of redirects,
- analytics,
- caching,
- database indexing,
- rate limiting,
- custom domains,
- QR codes,
- expiration,
- high availability.
